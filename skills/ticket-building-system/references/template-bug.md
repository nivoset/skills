# Bug Report Template

**Suggested tag:** `Bug`

May optionally link to a parent Feature, affected area, release, or component when the team uses that placement.

```markdown
## Title
[Short descriptive title of the bug]

## Summary
[One-line summary of the bug and its impact]

## Steps to Reproduce
1. [First step]
2. [Second step]
3. [Continue as needed]

## Current Behavior
[What actually happens]

## Expected Behavior
[What should happen instead]

## Severity
[Critical / High / Medium / Low]

## Notes
[Optional: screenshots, logs, related tickets, additional context]
```
